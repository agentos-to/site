auth_store.* ops — encrypted credential store reads.

The only SDK surface onto the encrypted credentials table. The
encryption key lives in the macOS Keychain; Python never holds it.

## `auth_store.read`

Read the best-scored decrypted credential row for a (domain, item_type, account?) tuple.

| Metadata | Value |
|---|---|
| **Audit log** | `request/item_type` → `item_type`, `request/domain` → `domain` |

### Request

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `domain` | `string` | **yes** |  | Canonical credential-store domain. |
| `item_type` | `string` | **yes** |  | Credential kind (login_credentials, cookie, oauth, api_key, …). |
| `account` | `string?` | no |  | Optional identifier disambiguator. |

### Response

| Field | Type | Description |
|---|---|---|
| `found` | `boolean` | Whether a matching row exists. |
| `identifier` | `string?` | Canonical identifier of the row. |
| `value` | `json?` | Decrypted JSON payload; shape depends on item_type. |
| `source` | `string?` | Where the row came from (onepassword, brave-browser, …). |
| `last_verified` | `i64?` | Unix timestamp of the most recent successful verification. |

### Notes

```text
Read one row from the encrypted credentials store. Returns the
decrypted JSON `value` plus identifier / source / last_verified.
Domain-normalization happens on the caller side — pass the
canonical credential-store form (e.g. `".approach.app"`).

Args:
    domain: Canonical credential-store domain.
    item_type: Credential kind (`"login_credentials"`, `"cookie"`, ...).
    account: Optional identifier disambiguator.

Returns:
    `{found, identifier?, value?, source?, last_verified?}`. On
    no-match, `found` is `false` and every other field is absent.
```
