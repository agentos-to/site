llm.* ops — schema resolution.

llm.resolve_tools walks apps' `provides` entries to build provider-format
tool definitions. Handler lives in agentos-core.

## `llm.resolve_tools`

Resolve app-scoped tool refs ("app.operation") to provider-native tool defs.

| Metadata | Value |
|---|---|
| **Service** | `llm` |

### Request

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `tools` | `string[]` | **yes** |  | Tool references (app.operation). |

### Response

| Field | Type | Description |
|---|---|---|
| `tools` | `json[]` | Resolved tool definitions, one per input ref (in order). |

### Notes

```text
Resolve a list of app-scoped tool refs (`"app.operation"`) into
provider-native tool definitions. Used by llm.agent loops that need
to pass the router's tool list into a chat call.

Args:
    tools: list of `"app.operation"` strings.

Returns:
    {tools: [tool_def, …]}.
```
